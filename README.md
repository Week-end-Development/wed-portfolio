## Nasza historia

Week-end Development powstalo podczas regularnych spotkan na Discordzie,
ktore poczatkowo koncentrowaly sie na wspolnej realizacji zadan akademickich.

W miare rozwoju wspolpracy naturalnie przeszlismy od pracy zaliczeniowej
do tworzenia autorskich projektow technologicznych realizowanych poza zajeciami.

Nazwa zespolu odzwierciedla ten model pracy - kluczowe etapy planowania,
implementacji i iteracji najczesciej odbywaly sie w wolne weekendy.

> Zamieniamy kofeine w ustrukturyzowana logike.

## Kim jestesmy dzisiaj

Jestesmy dwuosobowym zespolem studentow.
Laczymy backend, frontend i infrastrukture tak,
aby kazdy kolejny projekt byl lepszy od poprzedniego.

- OP_01 // Nikolas Feduniewicz // Spec_Backend
- OP_02 // Kacper Bos // Arch_AWS

## Jak pracujemy

1. Pomysl: rozbijamy problem na czesci i projektujemy architekture.
2. Prototyp: budujemy MVP i szybko testujemy zalozenia.
3. Iteracje: poprawiamy wydajnosc, UX i stabilnosc na podstawie feedbacku.

## Co jest teraz w toku

Rozwijamy kolejne projekty i przygotowujemy je do publikacji.
Publicznie pokazujemy to, co dowozi jakosc i ma sens dla uzytkownika.
